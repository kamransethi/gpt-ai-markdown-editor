# Security Policy

## Reporting a Security Vulnerability

If you discover a security vulnerability, **do not** open a public GitHub issue.

Instead, please report it responsibly:

### Option 1: GitHub Security Advisory (Recommended)
1. Go to [Security Advisories](https://github.com/kamransethi/gpt-ai-markdown-editor/security/advisories)
2. Click "Report a vulnerability"
3. Describe the vulnerability and steps to exploit it
4. Submit

GitHub will create a private draft advisory for discussion before public disclosure.

### Option 2: Direct Email
Email security details to: **kamran [at] example.com** (replace with your actual email)

Include:
- Description of the vulnerability
- Affected versions
- Steps to reproduce (if applicable)
- Potential impact
- Suggested fix (if you have one)

### What to Expect

1. **Acknowledgment** — We'll confirm receipt within 48 hours
2. **Assessment** — We'll evaluate the severity and impact
3. **Fix** — We'll work on a patch
4. **Disclosure** — We'll coordinate timing for public disclosure
5. **Credit** — We'll credit you in the security advisory (unless you prefer anonymity)

### Severity Guidelines

- **Critical** — Affects data security, user privacy, or system integrity
- **High** — Significant impact, easily exploitable
- **Medium** — Moderate impact, limited exploitability
- **Low** — Minor issue or difficult to exploit

## Security Best Practices

When using this extension:

- **Keep VS Code updated** — Security patches are released regularly
- **Report suspicious behavior** — If something seems off, let us know
- **Review dependencies** — Check `THIRD_PARTY_LICENSES.md` for included libraries
- **Use private repos for sensitive content** — Markdown files are stored locally, but use standard Git security practices

## Our Commitment

We take security seriously and commit to:

- Reviewing reports promptly
- Providing fixes for confirmed vulnerabilities
- Crediting discoverers (with permission)
- Maintaining transparency about security practices
- Following responsible disclosure timelines
