# Community Guidelines

Welcome to the Visual AI Markdown Editor community! 🎉

This guide explains how to engage with the project, report issues, suggest features, and contribute.

---

## Issues

We use **GitHub Issues** to track bugs, feature requests, and community discussions.

### Reporting a Bug

Found something broken? Here's how to report it:

1. **Check existing issues** — Search [issues](https://github.com/kamransethi/gpt-ai-markdown-editor/issues) to see if it's already reported
2. **Use the bug template** — Click "New Issue" → "Bug Report"
3. **Provide details**:
   - Clear description of the problem
   - Steps to reproduce
   - Expected vs. actual behavior
   - Your environment (OS, VS Code version, extension version)
   - Screenshots or screen recording if helpful
4. **Submit** — We'll triage and respond within 48 hours

### Requesting a Feature

Have an idea? We'd love to hear it.

1. **Check existing requests** — Search [issues](https://github.com/kamransethi/gpt-ai-markdown-editor/issues) first
2. **Use the feature template** — Click "New Issue" → "Feature Request"
3. **Explain**:
   - What problem does it solve?
   - How should it work?
   - Concrete use cases
   - Alternative solutions you considered
4. **Submit** — We'll review and discuss in the issue

### Discussions

For general questions, conversations, or topics that don't fit a bug or feature request:

- Use the **[Discussion](https://github.com/kamransethi/gpt-ai-markdown-editor/issues/new?template=discussion.md)** template
- Or visit **[GitHub Discussions](https://github.com/kamransethi/gpt-ai-markdown-editor/discussions)** (coming soon) for threaded conversations

## Labels Explained

We use labels to organize and prioritize issues:

### Type
- **bug** — Something isn't working
- **enhancement** — New feature or improvement
- **documentation** — Docs need updating
- **discussion** — Community conversation or question

### Priority (if assigned)
- **priority-high** — Urgent or high-impact
- **priority-medium** — Standard priority
- **priority-low** — Nice-to-have

### Status
- **status-in-progress** — Actively being worked on
- **status-blocked** — Waiting for something else
- **status-review** — Ready for review

### Community
- **good-first-issue** — Great for newcomers
- **help-wanted** — We're looking for contributions
- **wontfix** — Not planned to be fixed
- **duplicate** — Similar issue already exists

## Contributing Code

See [CONTRIBUTING.md](./CONTRIBUTING.md) for the development workflow, testing requirements, and commit conventions.

**Quick start for contributors:**
1. Fork the repo
2. Create a feature branch from `main`
3. Write tests with your code
4. Run `npm test` and `npm run build` to verify
5. Submit a PR with a clear description
6. We'll review and merge
